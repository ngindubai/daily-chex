Add JWT_SECRET to Render
RENDER ▸
Why: The auth system signs JWTs with a secret key. Without this env var, login will crash with "JWT_SECRET is not set".

When: After Step 3 (Auth) code is pushed.

  1. Open dashboard.render.com and log in
  2. Click daily-chex under your services list
  3. Click Environment in the left sidebar
  4. Click + Add Environment Variable
  5. In the Key field, type: JWT_SECRET
  6. In the Value field, paste a strong random string. Generate one by opening a terminal and running: 
    node -e "console.log(require('crypto').randomBytes(48).toString('base64'))"
    Copy the output and paste it as the value.
  7. Click Save Changes
  8. Render will auto-redeploy. Wait for the deploy to complete (green tick).
Verify: Open https://daily-chex.onrender.com/api/health — should return {"status":"ok"}. Then test login at the app homepage.

Reset Database — Re-seed with Hashed PINs
NEON ▸
Why: The old seed stored PINs as plain text. The new seed hashes them with bcrypt. Auth will fail unless the DB is re-seeded.

When: After Task 1 (JWT_SECRET) and after the Step 3 code is deployed.

Option A: Nuke & re-seed via Neon console

  1. Open console.neon.tech and log in
  2. Click your project (the one with ep-tiny-unit-ant8rdsp)
  3. Click SQL Editor in the left sidebar
  4. Paste and run this SQL to wipe all data: 
    TRUNCATE check_photos, check_items, checks, defects,
         check_template_items, check_templates,
         assets, people, teams, sites, companies
CASCADE;
  5. The next Render deploy will re-run seed-safe.js (it checks if companies table is empty). Trigger a manual deploy:
    • Go to Render → daily-chex → Manual Deploy → Deploy latest commit
  6. Wait for deploy to finish. Check logs for ✓ 6 people created.

Option B: Quick reset via terminal

  1. Open a terminal in the projects/chex/server directory
  2. Run with the Neon DATABASE_URL: 
    DATABASE_URL="postgresql://neondb_owner:npg_Zo4pGVf6weXE@ep-tiny-unit-ant8rdsp.c-6.us-east-1.aws.neon.tech/neondb?sslmode=require" npx tsx src/db/seed.ts
    ⚠ This will fail if data already exists (unique constraints). Use Option A first to TRUNCATE.
Verify: Go to https://daily-chex.onrender.com, select SAW Utilities, enter PIN 1234 → should log in as Mark Johnson (Manager).

Push Step 3 Code to GitHub
GITHUB ▸
Why: Render auto-deploys from the master branch. Push the auth code to trigger deployment.

When: After you've verified the build works locally (server tsc + app vite build both pass).

  1. Open a terminal in the projects/chex directory
  2. Stage all changes: 
    git add -A
  3. Commit: 
    git commit -m "Step 3: Auth system — PIN + email login, JWT, protected routes"
  4. Push to master: 
    git push origin master
  5. Render will auto-detect the push and start a new deploy. Watch the Render dashboard for a green tick.
Order matters: Do Task 1 (JWT_SECRET) before this push, otherwise the deploy will start without the secret and auth will break.
